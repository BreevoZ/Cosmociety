# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Cosmociety is a toy computational-physics research codebase: a 1D radial
temperature model of a self-gravitating body (center `r=0` to surface `r=1`)
that combines a core heating source, density-dependent heat capacity,
radiative diffusion from a toy opacity law, optional emergent convection, and
radiative surface cooling. The point of the project is that convective
regions are never prescribed — they emerge from local transport rules and are
detected by diagnostics after the fact. New physics should be added the same
way: one isolated, scannable effect at a time, not hard-coded regimes.

There is no packaged Python project (no `setup.py`/`pyproject.toml`) — always
run scripts from the repository root so `cosmociety`/`experiments` imports
resolve.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install numpy matplotlib pillow
```

No other runtime dependencies. There is currently no test suite, linter, or
CI config in this repo.

## Common commands

```bash
# Single case, quick non-equilibrium preview
python3 main.py --preview

# Single case, run to equilibrium (default case: baseline_envelope)
python3 main.py --case baseline_envelope

# All demo cases (see experiments/demo_cases.py for the named set)
python3 experiments/run_demo_cases.py                # preview mode
python3 experiments/run_demo_cases.py --equilibrium  # to equilibrium

# Parameter scans -> CSV under outputs/
python3 experiments/scan_convection.py --quick
python3 experiments/scan_convection.py
python3 experiments/scan_transport_law.py

# Text report from a scan CSV
python3 experiments/analyze_scan.py --input outputs/convection_scan.csv --output outputs/convection_scan_report.txt
```

`main.py` and `run_demo_cases.py` accept the same physics overrides
(`--convective-transport`, `--surface-cooling`, `--convective-max-diffusivity`,
`--opacity-temperature-power`, `--opacity-density-power`); see `main.py` for
the full list. Everything written under `outputs/` is git-ignored (only
`outputs/.gitkeep` is tracked) and is regenerated by these scripts.

## Architecture

Data flows through one big assembly function and one big result dict; nearly
every other module either feeds that function or consumes its output.

```
grid.py, profiles.py  -->  equilibrium.relax_to_equilibrium()  -->  result dict
                              (uses opacity.py, convection.py,          |
                               transport.py each step)                 v
                                                    diagnostics.py, visualize.py,
                                                    animation.py, experiments/*
```

- **`grid.py` / `profiles.py`** — pure functions building the static radial
  grid, core source, density profile, and initial temperature guess.
- **`opacity.py`** — toy opacity law `kappa = (rho/rho_surface)^q * T^-p` and
  the radiative diffusivity derived from it (`D_rad = D0 / kappa`, floored).
- **`convection.py`** — three independent convective *criteria*, each
  returning a cell-centered convective diffusivity plus a diagnostic "excess"
  array: `convective_diffusivity_from_gradient` (steep dT/dr), `..._from_radiative_demand`
  (luminosity radiation alone can't carry), `..._from_schwarzschild` (toy
  `nabla_rad > nabla_ad`). Also holds `convective_flux_from_adiabatic_excess`,
  the flux law used when `convective_transport="excess"`.
- **`transport.py`** — the actual explicit flux-form diffusion step
  (`diffusion_step`), supporting cartesian or spherical geometry, with
  radiative surface cooling (`~T_surface^4 - T_space^4`) applied as a boundary
  flux and center symmetry enforced (`T[0] = T[1]`).
- **`equilibrium.py`** — the orchestrator. `evaluate_convection` dispatches to
  whichever `convection.py` criterion is selected
  (`convective_criterion="gradient"|"radiative"|"schwarzschild"`, default
  `schwarzschild`); `relax_to_equilibrium` runs the explicit timestepping loop
  with an adaptive, CFL-limited `dt` (`stability_safety * dr^2 / max(D/C)`)
  until `max(|ΔT|) < tolerance` or `max_steps` is hit, then returns one large
  dict (temperature, source, density, pressure, all diffusivities/fluxes,
  history, per-step deltas/timesteps, `converged_step`, and every input
  parameter needed downstream). Treat this dict as the stable interface
  between simulation and everything else — diagnostics/visualize/animation
  and the experiment scripts all just read keys out of it.
- **`convective_transport`** has two modes with different physical meaning:
  `"excess"` (default) treats convection as transporting only the
  superadiabatic temperature drop (added as an extra interface *flux*, not a
  diffusivity, on top of radiative diffusion); `"diffusive"` instead adds the
  convective diffusivity directly to the radiative one and diffuses normally.
  The CFL `dt` bound always uses `radiative + convective` diffusivity even
  though `"excess"` mode applies the convective term as a flux rather than a
  diffusivity — this is intentional and still conservative, since the
  superadiabatic drop that flux transports can never exceed the raw
  temperature drop the same diffusivity would carry (see the comment above
  `stable_dt` in `relax_to_equilibrium`).
- **`diagnostics.py`** — turns a result dict into scalar summaries: contiguous
  convective region detection, `structural_regime` classification (e.g.
  `convective_core`, `convective_envelope`, `dual_convection`,
  `radiative_only`), and `regime` (structural regime prefixed with `open_` if
  the run didn't converge). This is what scan scripts group/report by.
- **`visualize.py` / `animation.py`** — static plots and relaxation GIFs.
  `animation.py` recomputes per-frame diffusivities and fluxes for each saved
  `history` snapshot by calling straight into `opacity.py`/`convection.py`/
  `transport.py` (rather than re-deriving the formulas locally), so it cannot
  silently drift from the static diagnostics/plots when a physics law changes.
- **`experiments/demo_cases.py`** — named parameter presets
  (`baseline_envelope`, `dual_convection`, `strong_envelope`, `no_convection`)
  consumed by `main.py` and `run_demo_cases.py`.
- **`experiments/scan_*.py`** — grid-search over `relax_to_equilibrium`
  parameters, writing one CSV row per run via `summarize_result`.
  `analyze_scan.py` reads that CSV back (independently, via `csv.DictReader`,
  not by importing the scan script) and builds a text report.

### Backward-compat parameter aliases

`opacity_power` / `radiative_density_power` are deprecated aliases for
`opacity_temperature_power` / `opacity_density_power`, still accepted by
`relax_to_equilibrium` and as hidden (`argparse.SUPPRESS`) CLI flags. When
threading new overrides through `main.py` / `run_demo_cases.py`, follow the
existing pattern of filtering `None` values before merging into `params` so
an unset override doesn't clobber a demo case's preset.
